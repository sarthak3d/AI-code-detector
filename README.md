Detect whether code was written by a human or generated by an LLM. Using an MLP meta-classifier trained on multi-model statistical features.

## How It Works

This detector extends [DetectCodeGPT](https://github.com/YerbaPage/DetectCodeGPT)'s zero-shot statistical approach into a supervised meta-classifier.

For each code sample, it runs **8 detection methods** across one or more LLMs and uses the resulting score vector as input to a trained MLP:

| Method | Description |
|---|---|
| **NPR** | Normalized Perturbation Ratio — compares log-rank before/after whitespace perturbation |
| **LRR** | Log-Likelihood Ratio of log-likelihood to log-rank |
| **LogRank** | Average log-rank of tokens under the model |
| **Entropy** | Average token-level prediction entropy |
| **Likelihood** | Average log-likelihood (negative cross-entropy loss) |
| **DetectGPT** | Curvature test — original [DetectGPT](https://github.com/eric-mitchell/detect-gpt) method |
| **T5-NPR** | NPR with T5 mask-fill perturbations instead of whitespace |
| **ID-NPR** | NPR with identifier-name masking (tree-sitter or regex fallback) |

The pipeline supports both **local HuggingFace models** (with 4-bit/8-bit quantization) and **API-hosted LLMs** (OpenAI, Anthropic, Google, any OpenAI-compatible endpoint).

## Quick Start

### Prerequisites

- Python 3.9+
- CUDA GPU recommended (CPU works but is slow for feature extraction)
- 8–64 GB VRAM depending on which detector models you use

Install dependencies:

```bash
pip install torch transformers accelerate bitsandbytes tqdm pandas scikit-learn optuna fastapi uvicorn
```

> [!NOTE]
> For API-based generation (`generate_api.py`), install the relevant provider SDK:
> `pip install openai anthropic google-generativeai`

## Full Training Pipeline

### Step 1 — Download Human Code Dataset

Downloads function-level code from [The Vault](https://github.com/FSoft-AI4Code/TheVault) for all 10 supported languages.

```bash
python download-dataset/download_dataset.py dataset_human --set function
```

**Options:**

| Flag | Default | Description |
|---|---|---|
| `DESTINATION_DIR` | — | Target directory for downloaded data |
| `--set` | `function` | Dataset split: `function`, `class`, `inline`, or `all` |

**Supported languages:** `python`, `javascript`, `java`, `c`, `cpp`, `go`, `rust`, `ruby`, `php`, `c_sharp`

---

### Step 2 — Generate AI Code

#### Option A: Local HuggingFace Models

Uses local models to complete function prompts extracted from the human dataset.

```bash
# Generate for all languages with a small model (fast)
python code-generation/generate.py \
    --path dataset_human \
    --language all \
    --model_name Salesforce/codegen-350M-mono \
    --output_dir dataset_ai

# Generate for specific languages with a larger model
python code-generation/generate.py \
    --path dataset_human \
    --language python,javascript,java \
    --model_name bigcode/starcoder2-7b \
    --output_dir dataset_ai \
    --max_num 1000
```

#### Option B: API-Hosted Models (Latest LLMs)

Use this to generate data from closed-source frontier models like GPT-4o, Claude 3.5 Sonnet, or Gemini 2.0 Flash. The output format is identical to Option A so the rest of the pipeline is unaffected.

```bash
# GPT-4o
export OPENAI_API_KEY=sk-...
python code-generation/generate_api.py \
    --provider openai --model gpt-4o \
    --path dataset_human --language all \
    --output-dir dataset_ai_gpt4o

# Claude 3.5 Sonnet
export ANTHROPIC_API_KEY=sk-ant-...
python code-generation/generate_api.py \
    --provider anthropic --model claude-3-5-sonnet-20241022 \
    --path dataset_human --language all \
    --output-dir dataset_ai_claude

# Google Gemini 2.0 Flash
export GOOGLE_API_KEY=AIza...
python code-generation/generate_api.py \
    --provider google --model gemini-2.0-flash \
    --path dataset_human --language python,java \
    --output-dir dataset_ai_gemini

# Any OpenAI-compatible endpoint (DeepSeek, Groq, Together, local vLLM)
python code-generation/generate_api.py \
    --provider openai-compat \
    --model deepseek-coder \
    --base-url https://api.deepseek.com/v1 \
    --api-key $DEEPSEEK_KEY \
    --path dataset_human --language python \
    --output-dir dataset_ai_deepseek

# List all supported providers
python code-generation/generate_api.py --list-providers
```

**API options:**

| Flag | Default | Description |
|---|---|---|
| `--provider` | `openai` | API provider: `openai`, `anthropic`, `google`, `openai-compat` |
| `--model` | provider default | Model name |
| `--max-num` | `1000` | Samples per language |
| `--concurrency` | `1` | Parallel API requests |
| `--temperature` | `0.2` | Sampling temperature |

---

### Step 3 — Extract Features

Runs all 8 detection methods across one or more local LLMs and writes a feature CSV.

```bash
# Fast run with small models (recommended for testing)
python generate-features/generate_features.py \
    --samples 500 \
    --models fast \
    --device cuda

# Full pipeline with all small + medium models
python generate-features/generate_features.py \
    --samples 500 \
    --models all-medium \
    --device cuda

# Target specific models
python generate-features/generate_features.py \
    --samples 500 \
    --models codegen-350m-multi,starcoder2-3b,deepseek-1.3b \
    --device cuda

# List all 20+ available detector models
python generate-features/generate_features.py --list-models
```

**Model groups:**

| Group | Models included |
|---|---|
| `fast` | codegen-350m-multi, starcoder2-3b, deepseek-1.3b |
| `all-small` | Models < 2 GB VRAM |
| `all-medium` | Models 2–8 GB VRAM |
| `all-large` | Models 8–16 GB VRAM |
| `all-xlarge` | Models 16–32 GB (8-bit quant) |
| `all-xxlarge` | Models > 32 GB (4-bit quant) |
| `all` | Every model in the registry |

Output CSVs are saved to `features/`. Each row is one code sample; columns are `{model}_{method}_score` for all 8 methods × N models, plus metadata columns.

---

### Step 4 — Train the MLP Detector

#### Initial training with Optuna hyperparameter optimization

```bash
# HPO with 50 trials (recommended)
python training/train_ANN.py \
    --features "features/*.csv" \
    --trials 50

# Quick smoke test (10 trials, 10-minute timeout)
python training/train_ANN.py \
    --features "features/*.csv" \
    --trials 10 --timeout 600

# Train with fixed hyperparameters (no Optuna)
python training/train_ANN.py \
    --features "features/*.csv" \
    --no-optimize
```

Trained models are saved to `dl_models/` as `.pkl` files containing the model weights, scaler, and feature names.

#### Updating the model for a new LLM (without retraining from scratch)

When you add data from a new model (e.g., a newly released LLM), use `update_ANN.py` to expand and fine-tune the existing MLP while preserving all previously learned weights.

```bash
# Standard update — transfer weights + fine-tune
python training/update_ANN.py \
    --model dl_models/mlp_model_20251214_184436.pkl \
    --features features/features_new.csv

# With warm-up phase: freeze old weights for 10 epochs first
python training/update_ANN.py \
    --model dl_models/mlp_model_20251214_184436.pkl \
    --features features/features_new.csv \
    --freeze-epochs 10 --lr 0.0005

# Full reinitalization (discard old weights, train from scratch)
python training/update_ANN.py \
    --model dl_models/mlp_model_20251214_184436.pkl \
    --features features/features_new.csv \
    --reinitialize
```

> [!TIP]
> Use `--freeze-epochs 10` when adding many new features at once. It lets the new input neurons warm up before full joint fine-tuning destabilizes the existing representations.

## Inference API

The `app/` directory contains a production-ready FastAPI service for real-time detection.

### Run locally

```bash
cd app
MLP_MODEL_PATH=../dl_models/mlp_model_<timestamp>.pkl \
DETECTOR_MODELS=deepseek-1.3b,starcoder2-3b \
uvicorn api.main:app --host 0.0.0.0 --port 8000
```

### Run with Docker

```bash
docker build -t ai-code-detector ./app

docker run --gpus all -p 8000:8000 \
  -e MLP_MODEL_PATH=/app/model/mlp_model.pkl \
  -e DETECTOR_MODELS=deepseek-1.3b \
  -v $(pwd)/dl_models:/app/model \
  ai-code-detector
```

### API endpoints

**`POST /evaluate`** — Detect whether code is AI-generated.

```bash
curl -X POST http://localhost:8000/evaluate \
  -H "Content-Type: application/json" \
  -d '{
    "code": "def binary_search(arr, target):\n    lo, hi = 0, len(arr) - 1\n    while lo <= hi:\n        mid = (lo + hi) // 2\n        if arr[mid] == target: return mid\n        elif arr[mid] < target: lo = mid + 1\n        else: hi = mid - 1\n    return -1",
    "language": "python"
  }'
```

**`GET /health`** — Check service status and loaded models.

### Environment variables

| Variable | Default | Description |
|---|---|---|
| `MLP_MODEL_PATH` | — | Path to the trained `.pkl` model file |
| `DETECTOR_MODELS` | `deepseek-1.3b` | Comma-separated detector model aliases |
| `DETECTOR_PERTURBATIONS` | `5` | Number of perturbations per sample |
| `MLP_THRESHOLD` | auto | Override the AI/human classification threshold |
